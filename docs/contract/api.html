Warning

  • In case of using filters, the meaning of "&" (query parameters separator) is defined according to the context, it can be AND or IN:
    1. **"AND"**, when the filter criterion is composed of a sequence of attributes that are different from each other.

      Example: ?a=1&b=2&c=3, should be

      *(((a = 1) AND (b = 2)) AND (c = 3))*

    2. **"IN"**, when the filter criterion is composed of a sequence of expressions applied to the same attribute. Whenever repeated parameters are found, they are grouped into an IN expression.

      Example: ?a=1&b=2&a=3, should be

      *a IN (1, 3) AND (b = 2)*

get /students

Get students filtered by specified criteria.

  • Request
  • Response

Headers

  • log_id: (string)

    Propagated identifier from top to bottom. Used to track errors or transactions

Query Parameters

  • eq_first_name: (string)

    Allows to filter by first_name with values matching to a specified parameter (non case sensitive).

  • eq_last_name: (string)

    Allows to filter by last_name with values matching to a specified parameter (non case sensitive).

  • eq_document_number: (string)

    Allows to filter by document_number with values matching to a specified parameter (non case sensitive).

  • eq_email: (string)

    Allows to filter by email with values matching to a specified parameter (non case sensitive).

  • gte_creation_date: (date-only)

    Allows to filter by creation_date with values greater than or equal to a specified date. Expected date format is yyyy-mm-dd.

  • lte_creation_date: (date-only)

    Allows to filter by creation_date with values less than or equal to a specified date. Expected date format is yyyy-mm-dd.

  • gender: (string - pattern: ((M | F),?)+)

    Allows to filter by gender listed in M | F.

  • view: (string - pattern: ((basic|full),?)+)

    Allows to ask for a simplified or detailed version of the response. Possible values are: basic|full

  • page: required(integer - default: 0 - minimum: 0)

    Page number to return. First page is always 1. If you specify a page number, you must specify page size.

  • page_size: (integer - default: 20 - minimum: 0)

    Items per page.

  • In case of using filters, the meaning of "&" (query parameters separator) is defined according to the context, it can be AND or IN:
    1. **"AND"**, when the filter criterion is composed of a sequence of attributes that are different from each other.

      Example: ?a=1&b=2&c=3, should be

      *(((a = 1) AND (b = 2)) AND (c = 3))*

    2. **"IN"**, when the filter criterion is composed of a sequence of expressions applied to the same attribute. Whenever repeated parameters are found, they are grouped into an IN expression.

      Example: ?a=1&b=2&a=3, should be

      *a IN (1, 3) AND (b = 2)*
